Detailed Description

This class stores information about a breakpoint on a specific source file for all processes. Each process can either have this breakpoint enabled or not. This is represented in the m_breakpoint_state array using the BreakpointState states.

This function sets a breakpoint for a process. For this to work the process MUST be in the stopped state. Otherwise GDB will not accept commands.

Parameters

Returns
true on success, false on error.
Note
Breakpoints are ID'd by a number which is set by GDB. It is saved in the m_numbers array per process. The m_numbers array is set by the set_number function which is called by the UIWindow::print_data_gdb function, when the corresponding GDB output has been parsed.

◆ update_states()

void Breakpoint::update_states ( )
private

This function resets the error states to their corresponding base state.

If a breakpoint could not be deleted it is in the ERROR_DELETING state but still existing, so reset to the CREATED state.

If a breakpoint could not be set it is in the ERROR_CREATING state but not existing, so reset to the NO_ACTION state.
